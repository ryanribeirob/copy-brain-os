## SEGMENTO 1: HOOK / ABERTURA (0:00 — ~0:60s em VSL)

**Filtros críticos: F1 + F7 + F2**

| # | Critério | ✓/✗ |
|---|----------|-----|
| H1 | O hook VIOLA a expectativa mais óbvia que o ICP tem sobre esse tipo de vídeo/apresentação | |
| H2 | O ICP específico se identifica em menos de 10 segundos | |
| H3 | Existe sinal de perda/ameaça OU curiosidade (não "bem-vindos ao nosso webinar") | |
| H4 | Cria uma promessa implícita do que vai ser revelado (gap preditivo aberto) | |
| H5 | Linguagem espelha o ICP, não linguagem corporativa | |

**Mínimo: 4/5** | H1 é inegociável — hook previsível = abandono imediato.

**Exemplos por tipo:**
- VSL de venda: "O que vou te mostrar nos próximos 20 minutos quebrou o que eu achava que sabia sobre [nicho]"
- Script de vendas: "Antes de falar sobre preço, preciso te mostrar algo que vai mudar como você vê esse investimento"
- Webinar: "Se você ficou até agora esperando eu te ensinar [promessa óbvia], este não é esse webinar"

---

## SEGMENTO 2: AGITAÇÃO DO PROBLEMA (min 1-5 em VSL típico)

**Filtros críticos: F2 + F3 + F5**

| # | Critério | ✓/✗ |
|---|----------|-----|
| P1 | A dor é nomeada com vocabulário EXATO do ICP (frases que ele usaria) | |
| P2 | Existe cena vivida do ICP no pior momento (não descrição do problema) | |
| P3 | A agitação PIORA progressivamente (não resolução prematura) | |
| P4 | A identidade do ICP é ameaçada (não só o bolso) | |
| P5 | O ICP pensa "isso é exatamente o que acontece comigo" | |

**Mínimo: 4/5** | P1 + P2 são inegociáveis.

**Sequência correta de agitação:**
1. Nomear a situação externa (o que acontece)
2. Pintar o impacto interno (como se sente)
3. Consequência de longo prazo (o que continua acontecendo se nada mudar)
4. Ameaça à identidade (quem ele está se tornando ao não resolver)

---

## SEGMENTO 3: REVELAÇÃO DO MECANISMO (min 5-10 em VSL típico)

**Filtros críticos: F7 + F6 + F3**

| # | Critério | ✓/✗ |
|---|----------|-----|
| M1 | A CAUSA do problema é revelada de forma não-óbvia (viola predição do ICP) | |
| M2 | O mecanismo tem um NOME específico (não "nossa metodologia") | |
| M3 | O nome do mecanismo é repetido 3+ vezes até o final do segmento | |
| M4 | É explicado POR QUÊ funciona antes de COMO funciona | |
| M5 | Existe analogia ou metáfora concreta que torna o mecanismo intuitivo | |
| M6 | Cria curiosidade sobre a próxima seção (gap preditivo #2 aberto) | |

**Mínimo: 4/6** | M1 + M2 são inegociáveis.

---

## SEGMENTO 4: PROVA SOCIAL / CASOS (min 10-15 em VSL típico)

**Filtros críticos: F2 + F3 + F4**

| # | Critério | ✓/✗ |
|---|----------|-----|
| PS1 | Pelo menos 1 caso com mesmo ICP (mesma situação de partida) | |
| PS2 | Caso inclui: dor antes + resultado depois + número + timeline + nome/cidade | |
| PS3 | A emoção do caso é descrita (não só o número) | |
| PS4 | Variabilidade de resultados presente (não todos iguais) | |
| PS5 | Pelo menos 1 caso supera objeção de "minha situação é diferente" | |

**Mínimo: 4/5** | PS2 completo é inegociável.

**Formato de caso que funciona:**
> "Dr. [Nome], [cidade], chegou para nós com [situação de partida específica]. Em [timeline], ele foi de [número antes] para [número depois]. O que mudou: [mecanismo aplicado]. Hoje ele [implicação de identidade]."

---

## SEGMENTO 5: OFERTA (min 15-18 em VSL típico)

**Filtros críticos: F4 + F6 + F4**

| # | Critério | ✓/✗ |
|---|----------|-----|
| O1 | Future pacing com cena completa (5 elementos) ANTES de revelar a oferta | |
| O2 | Stack de valor apresentado antes do preço (cada componente com valor unitário) | |
| O3 | Preço revelado com âncora de ROI: "investimento X, que se paga com Y procedimentos/mês" | |
| O4 | Garantia específica e inequívoca (não "satisfação garantida") | |
| O5 | A oferta é enquadrada como solução para a crença atualizada (não como produto genérico) | |
| O6 | Quick win de 7-14 dias especificado | |

**Mínimo: 5/6** | O3 + O4 são inegociáveis.

---

## SEGMENTO 6: CTA E FECHAMENTO (min 18-20 em VSL típico / min 40-60 em webinar)

**Filtros críticos: F7 + F4 + F6**

| # | Critério | ✓/✗ |
|---|----------|-----|
| C1 | CTA começa com "summary de crenças atualizadas" (o que o espectador aprendeu) | |
| C2 | A ação parece conclusão lógica do espectador, não pedido externo | |
| C3 | 1 único CTA — zero alternativas paralelas | |
| C4 | Próximo passo descrito em 1 sentença (o que acontece DEPOIS de clicar/ligar/preencher) | |
| C5 | Escassez real e específica presente | |
| C6 | Custo de inação mencionado em 1 linha | |

**Mínimo: 5/6** | C1 + C2 + C3 são inegociáveis.

## AVALIAÇÃO GERAL DO ROTEIRO

**Estrutura narrativa (F3 + F7):**

| Critério | ✓/✗ |
|----------|-----|
| O roteiro tem 1 big idea central (pode ser resumida em 1 frase) | |
| O arco emocional é progressivo (dor → esperança → desejo → certeza) | |
| O pico emocional máximo coincide com o momento de apresentar a oferta | |
| Cada transição de segmento tem "ganchos" que puxam para o próximo | |
| O nome do mecanismo está presente em pelo menos 5 momentos diferentes | |

## ERROS CRÍTICOS DE ROTEIRO (qualquer um = não gravar)

- [ ] Hook previsível (ICP consegue antecipar o que vem) → F1.3 error
- [ ] Agitação pula para solução antes de acumular dor suficientemente → F2.2 error
- [ ] Mecanismo sem nome específico → F3.3 error
- [ ] Prova social sem número específico + timeline → F3.4 error
- [ ] Preço antes de âncora de valor → F4.2 error
- [ ] Garantia vaga → F4.3 error
- [ ] 2+ CTAs no fechamento → F4.4 error

## CHECKLIST DE ENTREGA (antes de gravar/publicar)

- [ ] Roteiro auditado passa nos scores mínimos
- [ ] Presenter fez 1 ensaio completo em voz alta
- [ ] Momentos de ÊNFASE marcados no script (caps, sublinhado)
- [ ] Pausas estratégicas marcadas (antes de revelar número importante)
- [ ] Tons de voz por segmento definidos:
  - Hook: energia alta, curioso
  - Agitação: empático, próximo
  - Mecanismo: confiante, revelador
  - Prova: orgulhoso, honesto
  - Oferta: calmo, certo
  - CTA: decisivo, sem pressão
