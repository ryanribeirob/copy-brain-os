## BLOCO 1: HOOK/HEADLINE (F1 + F7)

| # | Critério | ✓/✗ |
|---|----------|-----|
| 1.1 | Não consigo "completar" a headline antes de lê-la (novelty) | |
| 1.2 | O ICP específico se identifica em <3 segundos | |
| 1.3 | Existe perda/ameaça ou sinal de "está perdendo algo" | |
| 1.4 | A headline viola a predição mais óbvia do ICP sobre o segmento | |
| 1.5 | Existe incongruência que PROMETE resolução (não deixa no ar) | |

**Mínimo:** 4/5 ✓ | Se <4: reescrever o hook ANTES de qualquer outra coisa.

---

## BLOCO 2: CORPO / PROBLEMA / SOLUÇÃO (F2 + F3 + F5)

| # | Critério | ✓/✗ |
|---|----------|-----|
| 2.1 | A dor usa as PALAVRAS EXATAS que o ICP usa (não eufemismo corporativo) | |
| 2.2 | Existe pelo menos 1 história com personagem + arco + número específico | |
| 2.3 | Before/after é VIVIDO como cena (não descrito abstratamente) | |
| 2.4 | Cada afirmação abstrata tem imagem concreta equivalente | |
| 2.5 | Existe pelo menos 1 momento de linguagem sensorial (físico/visceral) | |
| 2.6 | A causa do problema é apresentada de forma não-óbvia (viola predição) | |
| 2.7 | Sentenças variam de curtas a longas (não monotonia) | |

**Mínimo:** 5/7 ✓ | Falha em 2.1 = crítico independente dos outros.

---

## BLOCO 3: OFERTA / PROVA / CTA (F4 + F6 + F7)

| # | Critério | ✓/✗ |
|---|----------|-----|
| 3.1 | Âncora de ROI/valor apresentada ANTES do preço/investimento | |
| 3.2 | Prova social é do MESMO ICP com nome + resultado + timeline | |
| 3.3 | Future pacing com cena específica (tempo + lugar + evento + sentimento) | |
| 3.4 | Quick win nos primeiros 7-14 dias mencionado | |
| 3.5 | Garantia elimina completamente o risco de erro | |
| 3.6 | CTA é 1 ação única e clara (sem alternativas paralelas que criam paralisia) | |
| 3.7 | CTA parece conclusão lógica do ICP, não imposição externa | |
| 3.8 | Escassez presente é real e verificável (não "oferta expira hoje") | |

**Mínimo:** 6/8 ✓ | Falha em 3.1 + 3.5 juntos = crítico.

---

## BLOCO 4: CLAREZA COGNITIVA (F4 + F5)

| # | Critério | ✓/✗ |
|---|----------|-----|

| 4.2 | Seções de CTA/pricing têm frases curtas (<15 palavras) | |
| 4.3 | Zero jargão não explicado | |
| 4.4 | Máximo 1 ideia central por parágrafo | |
| 4.5 | Nenhum ponto em que pensei "mas que que isso quer dizer?" | |

**Mínimo:** 4/5 ✓

## ERROS CRÍTICOS IMEDIATOS (qualquer um desses = não publicar)

- [ ] Headline que o ICP pode "completar" antes de ler → F1.3 error
- [ ] Dor em linguagem corporativa, não na voz do ICP → F2.1 error
- [ ] Preço antes de âncora de valor → F4.2 error
- [ ] CTA com 2+ opções paralelas sem hierarquia → F4.4 error
- [ ] Garantia vaga ("satisfação garantida") → F4.3 error
- [ ] Prova social anônima ou sem número específico → F3.4 error

---

## MELHORIAS RÁPIDAS (10 minutos de reescrita para ganho máximo)

1. **Substitua o hook** por versão que viola a predição mais óbvia do ICP
2. **Adicione 1 cena de before/after** vivida (não descrita) na agitação
3. **Mova o ROI** para antes do preço (se estiver depois)
4. **Adicione quick win de 7-14 dias** na promessa
5. **Simplifique o CTA** para 1 ação com próximos passos em 1 sentença
